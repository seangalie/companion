# Support

## How to Get Help

This project is maintained by a small team of volunteers. We do our best to respond to issues and questions in a timely manner, but please keep in mind that support is provided on a best-effort basis.

Before reaching out, please:

- **Search existing issues** to see if your question or problem has already been addressed.
- **Check the documentation** in the repository, including the [README](README.md) and [CHANGELOG.md](CHANGELOG.md) for recent changes that may be relevant.

## Asking Questions

The best way to ask a question is to [open a Discussion](https://github.com/seangalie/companion/discussions) or [open an Issue](https://github.com/seangalie/companion/issues/new). Please use the appropriate issue template if one is available.

When asking a question, please include:

- A clear description of the problem or question.
- Steps to reproduce the issue (if applicable).
- Relevant environment details (e.g., OS, runtime version, etc.).

## Reporting Bugs

To report a bug, please [open an Issue](https://github.com/seangalie/companion/issues/new) and include:

1. A clear and descriptive title.
2. Steps to reproduce the behavior.
3. The expected behavior.
4. The actual behavior you observed.
5. Any relevant logs, screenshots, or error messages.

Please search existing issues before filing a new one to avoid duplicates.

## Requesting Features

Feature requests are welcome! Please [open an Issue](https://github.com/seangalie/companion/issues/new) and describe:

- The problem you are trying to solve.
- Your proposed solution or feature.
- Any alternatives you have considered.

## Reporting Security Vulnerabilities

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, follow the guidelines in our [Security Policy](SECURITY.md) to report vulnerabilities responsibly.

## Supported Versions

For information about which versions are currently supported, please refer to the [Security Policy](SECURITY.md).

## Contributing

We welcome contributions! If you would like to contribute, please:

1. Fork the repository.
2. Create a feature branch for your changes.
3. Submit a pull request with a clear description of your changes.

By contributing, you agree to abide by the [Code of Conduct](CODE_OF_CONDUCT.md).

## Contact

For general inquiries, you can reach the maintainer through the contact form at [https://www.seangalie.com](https://www.seangalie.com/contact/).
