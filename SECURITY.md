# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please report it responsibly.

**Please do not open public issues for security vulnerabilities.**

Instead, report security issues by emailing the maintainer directly through the secure contact form at [https://www.seangalie.com](https://www.seangalie.com/contact/).

Please include the following details in your report:

- A clear description of the vulnerability.
- Steps to reproduce the issue.
- The potential impact or severity.
- Any suggested fixes or mitigations (optional).

You can expect an initial response within *5 business days*. We will work with you to verify the issue, determine its impact, and develop a fix. Once resolved, we will coordinate disclosure and credit you unless you prefer to remain anonymous.

## Security Best Practices

When using this template, consider the following:

- Keep dependencies up to date.
- Avoid committing sensitive data such as API keys, passwords, or tokens.
- Review `.gitignore` and `.gitattributes` rules to ensure they meet your project's needs.
- Enable branch protection and require code review for changes.
